SECTION 1B — HOW TO READ TENNISEXPLORER HTML (MANDATORY)

CRITICAL: TennisExplorer HTML always contains Pinnacle. NEVER report Pinnacle as absent without first searching the full HTML text for the string "Pinnacle". If you cannot find it, state explicitly that you searched and it was not present.

Step 1 — Extract all bookmaker current odds

In the HTML, find the "Betting odds" / "Home/Away" section. It contains a table with one row per bookmaker. Each bookmaker row has:

  • Bookmaker name (e.g. "Pinnacle", "bet365", "1xBet")
  • Current FAV odds (most recent timestamp, first in the list)
  • Current UND odds (most recent timestamp, first in the list)
  • Historical snapshots with timestamps: format DD.MM. HH:MM | odds | delta
  • Opening odds (labeled "Opening odds" at the bottom of each cell)

Step 2 — Build Pinnacle price history for morfologia

From the Pinnacle row, extract FAV odds chronologically (oldest = Opening odds at bottom → newest = first entry):

Opening odds → snapshot N-1 → ... → snapshot 2 → snapshot 1 → current odds
Example: 1.69 → 1.72 → 1.75 → 1.81 → 1.85

Read the SHAPE of this sequence to classify morfologia:

  • Continuous rise: S (PRO UND strong)
  • Continuous descent: D (EV — check DB)
  • Descent then flat cap: DC (PRO UND)
  • Rise then flat: SF (EV)
  • Up then down: G (PRO UND)
  • Down then up: VD (EV)
  • Flat throughout: P (EV)
  • Abrupt jump at one point: ST (nd)

Step 3 — Extract current odds for ALL bookmakers

For each bookmaker in the HTML, record the MOST RECENT (first listed) odds for FAV side and UND side. Build two lists:

FAV side current odds: [Pinnacle: X.XX, bet365: X.XX, 1xBet: X.XX, ...]
UND side current odds: [Pinnacle: X.XX, bet365: X.XX, 1xBet: X.XX, ...]

Step 4 — Calculate outlier on FAV side

  1. Sort all FAV odds descending
  2. Find the highest FAV odds among ALL bookmakers
  3. If Pinnacle FAV odds = highest (or tied for highest) → Outl_FAV = SI → PRO UND
  4. If Pinnacle FAV odds is NOT the highest → Outl_FAV = nd

Step 5 — Calculate outlier on UND side

  1. Sort all UND odds descending
  2. Find the highest UND odds among ALL bookmakers
  3. If Pinnacle UND odds = highest (or tied for highest) → Outl_UND = SI → PRO FAV
  4. If Pinnacle UND odds is NOT the highest → Outl_UND = nd
If Pinnacle is highest on BOTH FAV and UND sides simultaneously → both = nd (signals cancel).
If Pinnacle is NOT highest on either side → both = nd.

SECTION 2 — KEY DEFINITIONS

  • FAV = player with lowest Pinnacle odds (always)
  • UND = player with highest Pinnacle odds (always)
  • HOME/AWAY = graphic convention only, not field advantage
  • Esito in DB = always from FAV perspective: W = FAV wins, L = FAV loses
  • Outl_UND = Pinnacle highest on UND side → PRO FAV
  • Outl_FAV = Pinnacle highest on FAV side → PRO UND

SECTION 5 — SIGNAL 3: OLS MODEL (optional)

Pipeline — execute in exact order

Step 1: prob_subject = (1/q_subject) / (1/q_subject + 1/q_opponent)
Step 2: x = ln(rank_opponent) [>1500 → use 1500] | y = ln(prob_subject)
Step 3: a = SUM[(xi-x_mean)(yi-y_mean)] / SUM[(xi-x_mean)^2] | b = y_mean - a*x_mean
Step 4: R² = 1 - SS_res/SS_tot
Step 5: prob_forecast = exp(a * ln(rank_today) + b) | quota_ff = 1/prob_forecast
Step 6: Δ% = (quota_ff - quota_Pinnacle_subject) / quota_Pinnacle_subject * 100
        Δ% positive → PRO subject | Δ% negative → PRO opponent
Step 7: Check extreme ranks — verify R² with and without
NEVER invert the Δ% signal rule. Apply mechanically always.

SECTION 8 — DATASET RULES

  1. Min N: 8. Window: 12 months (extend to 18 if N < 8)
  2. Duplicates same rank → remove oldest
  3. Rank > 1500 → use 1500
  4. Extreme ranks: check R² with and without, do NOT remove automatically
  5. Exclude qualifying if main draw and vice versa
  6. Hard floor: odds below 1.80 excluded (override documented)
  7. Rank filter ±200 → if N < 8 → ±400 → if still < 8 → all data
  8. If N < 8 with all data → STOP
  9. Cherry picking forbidden
