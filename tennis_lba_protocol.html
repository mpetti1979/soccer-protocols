SECTION 5 — SIGNAL 3: OLS MODEL (optional)

Pipeline — execute in exact order

Step 1 — No-vig probability

prob_subject = (1/q_subject) / (1/q_subject + 1/q_opponent)

Step 2 — Log transformation

x = ln(rank_opponent)   [rank >1500 → use 1500]
y = ln(prob_subject)

Step 3 — OLS regression

x_mean = mean of all x values
y_mean = mean of all y values
a = SUM[(xi - x_mean)(yi - y_mean)] / SUM[(xi - x_mean)^2]
b = y_mean - a * x_mean

Step 4 — R²

SS_res = SUM[(yi - ŷi)^2]   where ŷi = a*xi + b
SS_tot = SUM[(yi - y_mean)^2]
R² = 1 - SS_res / SS_tot

Step 5 — Fair forecast

ln(prob_forecast) = a * ln(rank_opponent_today) + b
prob_forecast = exp(ln(prob_forecast))
quota_fair_forecast = 1 / prob_forecast

Step 6 — Δ%

Δ% = (quota_fair_forecast - quota_Pinnacle_subject) / quota_Pinnacle_subject * 100
  • Δ% positive → fair_forecast > Pinnacle → PRO subject
  • Δ% negative → fair_forecast < Pinnacle → PRO opponent
NEVER invert the signal rule. Apply mechanically always.

Step 7 — Extreme rank check

  • If dataset contains rank very far from mean → verify R² with and without
  • Do NOT remove automatically → choose version with more stable R²
  • Always document which version is used

SECTION 8 — DATASET RULES (OLS model)

  1. Minimum N: 8 observations
  2. Window: last 12 months. If N < 8 → extend to 18 months
  3. Duplicates: same rank → remove oldest
  4. Extreme ranks: do NOT remove automatically → check R² with and without
  5. Rank > 1500: replace with 1500 before ln()
  6. Exclude qualifying if match is main draw and vice versa
  7. Hard floor: odds below 1.80 excluded (override documented case by case)
  8. Rank filter today ±200. If N < 8 → ±400 → if still < 8 → all data
  9. If N < 8 with all data → STOP before proceeding
  10. Cherry picking forbidden: do not substitute data to improve R²
